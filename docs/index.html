你看不到的问题，正在拖慢你的 Agent

社区实测：8-10 个 Skill 就开始互相冲突。你装了多少个？

💰

隐性 Token 税

Plugin 的 description 每次对话都注入，你看不到也关不掉。通常占总注入量的 74%。

✂

静默截断

超过预算时，每个 description 被砍到 250 字符。超出部分的 trigger 关键词永久丢失。

⚠

Hook 冲突

多个插件注册同一个 matcher，每次工具调用触发多个 hook，增加延迟和不确定性。

🚫

零使用可见性

官方不提供 Skill 使用频率追踪。GitHub issue #35319 已提出，至今无回应。

一次扫描，全面诊断

/skill-monitor security

安全扫描

CRITICAL / HIGH / MEDIUM 三级告警。检测反弹 Shell、凭据泄露、敏感路径访问、未经确认的配置修改。

/skill-monitor cost

Token 成本审计

精确提取每个 Skill 的 description（支持单行和多行 YAML）。User / Plugin 分项统计，Top 10 成本排行。

/skill-monitor budget

描述预算诊断

基于官方文档：1% context window 预算、250 字符截断上限、Skill 数量对匹配准确性的影响评估。

/skill-monitor hooks

Hook 冲突检测

扫描 settings.json 和 Skill frontmatter，找出注册了相同 matcher 的重复 hook。

/skill-monitor overlap

跨层重叠检测

LLM 驱动：对比 user skill 和 plugin skill 的描述，找出功能重复项，建议保留或移除。

/skill-monitor compress

描述压缩优化

将超长 description 压缩到 250c 以内，确保 trigger 关键词在可见窗口中。Front-load 最佳实践。
